# Editing Skill

Professional editing methodology for transforming rough drafts into polished prose.

## When to Use This Skill

Use this skill when the user asks to:
- Proofread or check text for errors
- Improve clarity or readability
- Polish or refine written content
- Make text more concise
- Adapt tone or style
- Review and suggest improvements

## Core Principle: Preserve Voice, Improve Execution

Good editing improves the writing without erasing the author's voice. Ask: "What did they mean to say?" then help them say it better.

## The Three Editing Layers

Edit in passes, from big picture to fine detail:

### Layer 1: Structural Edit (Big Picture)

Focus on organization and flow:

| Check | Questions to Ask |
|-------|------------------|
| **Purpose** | Is the main point clear in the first paragraph? |
| **Structure** | Does the organization make sense? |
| **Completeness** | Is anything missing? Anything unnecessary? |
| **Flow** | Do sections connect logically? |
| **Opening** | Does it hook the reader? |
| **Closing** | Is there a clear conclusion/takeaway? |

**Common structural issues:**
- Buried lede (main point hidden deep in text)
- Missing context (jumping in without setup)
- Redundant sections (saying same thing twice)
- Weak transitions (sections feel disconnected)
- Anti-climax (important points at the end)

### Layer 2: Line Edit (Sentence Level)

Focus on sentences and paragraphs:

| Check | Questions to Ask |
|-------|------------------|
| **Clarity** | Is each sentence easy to understand on first read? |
| **Concision** | Can any words be cut without losing meaning? |
| **Flow** | Do sentences connect smoothly? |
| **Rhythm** | Is there variety in sentence length? |
| **Voice** | Is the tone consistent throughout? |
| **Strength** | Are verbs active and specific? |

**Common line-level issues:**
- Run-on sentences (too much packed in one)
- Choppy prose (all sentences same length)
- Unclear antecedents ("it" or "this" without clear reference)
- Passive voice overuse
- Nominalization (verbs turned into nouns)

### Layer 3: Copy Edit (Mechanics)

Focus on correctness:

| Check | What to Look For |
|-------|------------------|
| **Grammar** | Subject-verb agreement, tense consistency |
| **Punctuation** | Commas, semicolons, dashes used correctly |
| **Spelling** | Typos, homophones (their/there/they're) |
| **Consistency** | Style choices applied throughout |
| **Formatting** | Headings, lists, paragraphs consistent |

## Common Fixes

### Wordiness Reductions

| Wordy | Concise |
|-------|---------|
| in order to | to |
| due to the fact that | because |
| at this point in time | now |
| in the event that | if |
| has the ability to | can |
| with regard to | about |
| in spite of the fact that | although |
| for the purpose of | to |
| make a decision | decide |
| give consideration to | consider |
| is able to | can |
| in the near future | soon |
| a large number of | many |
| the majority of | most |
| in close proximity to | near |

### Passive to Active Voice

| Passive (Weak) | Active (Strong) |
|----------------|-----------------|
| The report was written by the team | The team wrote the report |
| Mistakes were made | We made mistakes |
| The decision was reached | The committee decided |
| It was determined that | We determined |
| The meeting will be held | We'll hold the meeting |

### Weak to Strong Verbs

| Weak | Strong |
|------|--------|
| make a decision | decide |
| have a discussion | discuss |
| give an explanation | explain |
| do an analysis | analyze |
| make an improvement | improve |
| take into consideration | consider |
| put emphasis on | emphasize |
| is indicative of | indicates |

### Hedging Cleanup

| Over-hedged | Direct |
|-------------|--------|
| I think perhaps maybe | Consider |
| It would seem that | It seems / [Just state it] |
| It could be argued that | [State the argument] |
| In my opinion, I believe | [State the opinion] |
| It is worth noting that | [Just note it] |

### Filler Word Removal

Usually delete:
- very, really, quite, rather
- just, actually, basically, literally
- definitely, certainly, absolutely
- kind of, sort of, somewhat
- in fact, as a matter of fact
- it should be noted that
- it is important to note

## Tone Adjustments

### Too Formal → Conversational
- "We would like to inform you" → "Here's what you need to know"
- "Please do not hesitate to contact" → "Feel free to reach out"
- "It has come to our attention" → "We noticed"
- "Attached please find" → "I've attached"

### Too Casual → Professional
- "Hey, so basically..." → "The key point is..."
- "This thing is super important" → "This is critical"
- "We're gonna do this" → "We plan to"
- "Totally awesome" → "Highly effective"

### Too Technical → Plain Language
- "Utilize" → "Use"
- "Leverage" → "Use" or "Build on"
- "Optimize" → "Improve"
- "Implement" → "Start" or "Put in place"
- "Facilitate" → "Help" or "Enable"
- "Paradigm" → "Model" or "Approach"

## Style Guides Quick Reference

### Default Standards
- **Spelling**: American English (unless specified)
- **Comma**: Oxford comma (a, b, and c)
- **Periods**: One space after
- **Numbers**: Spell out under 10; use digits for 10+
- **Dates**: January 15, 2025 (not 15 January 2025)
- **Time**: 3:00 PM (not 3pm or 15:00)

### Common Style Choices
| Element | Options |
|---------|---------|
| **Serial comma** | "a, b, and c" (Oxford) vs "a, b and c" |
| **Capitalization** | Title Case vs Sentence case for headers |
| **Contractions** | "don't" vs "do not" (tone-dependent) |
| **First person** | "I/we" vs "the author/the company" |

## Editing Checklist

### Before Starting
- [ ] Understand the purpose and audience
- [ ] Note any style guide requirements
- [ ] Read through once without editing

### Structural Pass
- [ ] Main point clear in opening
- [ ] Logical section order
- [ ] No redundant sections
- [ ] Smooth transitions
- [ ] Strong conclusion

### Line Edit Pass
- [ ] Each sentence clear on first read
- [ ] Active voice where appropriate
- [ ] Varied sentence length
- [ ] No unclear pronouns
- [ ] Consistent tone

### Copy Edit Pass
- [ ] No spelling errors
- [ ] Grammar correct
- [ ] Punctuation correct
- [ ] Formatting consistent
- [ ] Style guide followed

### Final Check
- [ ] Read aloud for rhythm
- [ ] Check all proper nouns
- [ ] Verify facts and numbers
- [ ] Confirm links work (if applicable)

## Editing Principles

1. **Read first, edit second** - Understand before changing
2. **Preserve meaning** - Never change what the author meant
3. **Respect voice** - Edit for clarity, not to impose your style
4. **Be surgical** - Change only what needs changing
5. **Explain choices** - Help the author learn
6. **Question, don't assume** - Flag unclear meaning rather than guessing
7. **Know when to stop** - Don't over-edit

## When NOT to Edit

- **Direct quotes** - Never change quoted material
- **Brand names** - Match official capitalization
- **Technical terms** - Verify before "fixing"
- **Intentional style** - Poetry, dialogue, stylistic choices
- **Cultural expressions** - Idioms, regional language
